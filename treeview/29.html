Supernetting in Network Layer คืออะไร/h3>

Supernetting (หรือ Route Aggregation หรือ Prefix Aggregation) เป็นเทคนิคในการจัดการกับเส้นทางในระบบเครือข่าย โดยการรวมหลายๆ เครือข่ายย่อย (subnetworks) ที่มีความคล้ายคลึงกันเข้าด้วยกันเป็นเครือข่ายใหญ่ (supernetwork หรือ supernet) ซึ่งช่วยลดขนานข้อมูลในตารางเส้นทาง (routing table) และทำให้การตัดสินใจเกี่ยวกับเส้นทางง่ายขึ้น รวมถึงลดการโฆษณาเส้นทางไปยังเกตเวย์ที่อยู่ใกล้เคียง12.

ความสำคัญของ Supernetting:

  • ลดขนานข้อมูลในตารางเส้นทาง: การรวมเครือข่ายย่อยเป็นเครือข่ายใหญ่ช่วยลดขนานข้อมูลในตารางเส้นทาง ซึ่งทำให้การค้นหาเส้นทางเป็นไปอย่างมีประสิทธิภาพมากขึ้น
  • ลดขนานข้อมูลในการโฆษณาเส้นทาง: การรวมเครือข่ายย่อยเป็นเครือข่ายใหญ่ลดขนานข้อมูลในการโฆษณาเส้นทางไปยังเกตเวย์ที่อยู่ใกล้เคียง ทำให้เครือข่ายมีประสิทธิภาพมากขึ้น

ข้อควรระวังในการ Supernetting:

  • ทุกเครือข่ายย่อยควรเป็นเครือข่ายที่ต่อเนื่องกัน
  • ขนานของเครือข่ายย่อยทุกเครือข่ายควรเป็นขนานเท่ากันและเป็นเลขฐาน 2^n
  • ที่อยู่ IP แรกควรหารด้วยขนานของเครือข่ายย่อยทั้งหมดลงตัว
  • ตัวอย่าง: สมมติมีเครือข่ายย่อย 4 ตัวของคลาส C: 200.1.0.0, 200.1.1.0, 200.1.2.0, 200.1.3.0 สร้างเครือข่ายใหญ่ที่มี Network Id เดียว ก่อน Supernetting ตารางเส้นทางจะมีดังนี้:
Network Id Subnet Mask Interface
200.1.0.0 255.255.255.0 A
200.1.1.0 255.255.255.0 B
200.1.2.0 255.255.255.0 C
200.1.3.0 255.255.255.
